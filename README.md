## üì± Overview

MetadataKill is a privacy-focused iOS/iPadOS application that removes ALL metadata from photos and videos without compromising quality. All processing happens locally on your device‚Äîno data ever leaves your iPhone or iPad.

### üéØ Key Features

#### Image Processing
- ‚úÖ **Complete Metadata Removal**: EXIF, IPTC, XMP, GPS, color profiles, thumbnails
- ‚úÖ **Format Support**: JPEG, HEIC/HEIF, PNG, WebP, RAW/DNG (converts to JPEG)
- ‚úÖ **Orientation Baking**: Corrects image rotation by redrawing pixels
- ‚úÖ **Color Space Management**: Optional sRGB conversion from Display P3
- ‚úÖ **PNG Chunk Removal**: Strips tEXt, iTXt, zTXt chunks
- ‚úÖ **Quality Control**: Adjustable JPEG/HEIC quality settings
- ‚úÖ **HEIC to JPEG Conversion**: Optional format conversion

#### Video Processing
- ‚úÖ **Fast Re-muxing**: No re-encoding for maximum speed (default)
- ‚úÖ **Smart Processing**: Automatic fallback to re-encoding if metadata persists
- ‚úÖ **Safe Re-encoding**: H.264/AAC output with complete metadata removal
- ‚úÖ **QuickTime Metadata**: Removes udta, ISO6709 location, chapters, timecode
- ‚úÖ **HDR Support**: Optional HEVC 10-bit preservation
- ‚úÖ **Validation**: Post-processing checks for duration, tracks, and metadata

#### Live Photos & Special Formats
- ‚úÖ **Live Photo Support**: Processes both image and video components
- ‚úÖ **Burst Photos**: Handles multiple sequential images
- ‚úÖ **Depth/Portrait**: Cleans depth map metadata
- ‚úÖ **iCloud Integration**: Automatic download when needed

#### User Experience
- ‚úÖ **SwiftUI Interface**: Native, responsive design
- ‚úÖ **Batch Processing**: Handle multiple files with progress tracking
- ‚úÖ **Share Extension**: Clean and share directly from Photos app
- ‚úÖ **Siri Shortcuts**: "Clean my last photo/video"
- ‚úÖ **Background Tasks**: Continue processing in background
- ‚úÖ **Accessibility**: VoiceOver, Dynamic Type, high contrast support
- ‚úÖ **Localization**: English and French (automatically detects device language)

### Module Structure

Pure Swift business logic with zero dependencies:
- **Models**: `MediaItem`, `CleaningSettings`, `CleaningResult`, `MetadataType`

- **Repositories**: Interface definitions for data access
- **Errors**: Comprehensive error handling

#### Data (`Sources/Data`)
Implementation of data processing:
- **ImageProcessing**: `ImageMetadataCleaner` using CoreGraphics/ImageIO
- **VideoProcessing**: `VideoMetadataCleaner` using AVFoundation
- **Storage**: Local file system operations
- **UseCases**: Concrete implementations

#### Platform (`Sources/Platform`)
iOS-specific integrations:
- **PhotoKit**: Photo library access
- **FileSystem**: UIDocumentPicker, Drag & Drop

#### App (`Sources/App`)
SwiftUI interface:
- **Views**: Home, ImageCleaner, VideoCleaner, Settings
- **ViewModels**: Observable objects with @MainActor
- **DI**: Dependency injection container

## üìñ Usage

### Basic Workflow

1. **Select Media**:
   - Choose photos/videos from library
   - Import from Files app
   - Use drag & drop (iPad)

2. **Configure Settings** (optional):
   - Access via Settings gear icon
   - Adjust quality, processing mode, etc.

3. **Clean Metadata**:
   - Tap "Clean Photos" or "Clean Videos"
   - Monitor progress
   - Review results

4. **Access Cleaned Files**:
   - Files saved to: Documents/MetadataKill_Clean/
   - Original files remain untouched (by default)
   - Option to share directly

### Settings Reference

#### Metadata Removal
- **Remove GPS**: Strip location data (always recommended)
- **Remove All Metadata**: Remove everything (enabled by default)

#### File Options
- **Preserve File Date**: Keep original modification time
- **Output Mode**: Replace/New Copy/New Copy with Timestamp

#### Image Settings
- **Convert HEIC to JPEG**: Format conversion option
- **HEIC Quality**: 50-100% (default: 85%)
- **JPEG Quality**: 50-100% (default: 90%)
- **Force sRGB Color**: Convert Display P3 to sRGB
- **Bake Orientation**: Correct rotation in pixels

#### Video Settings
- **Processing Mode**:
  - Fast Copy: Re-mux without re-encoding (fastest)
  - Safe Re-encode: Always re-encode (most thorough)
  - Smart Auto: Try fast, fallback to re-encode (recommended)
- **Preserve HDR**: Use HEVC 10-bit for HDR content

#### Performance
- **Thermal Monitoring**: Pause on overheating
- **Concurrent Operations**: 1-8 parallel tasks

## üêõ Known Limitations

### Image Processing
- **RAW/DNG**: Converted to JPEG (lossy)
- **WebP**: Support depends on iOS version
- **Animated Images**: First frame only

### Video Processing
- **DRM Content**: Cannot process protected videos
- **Corrupt Files**: Detected and skipped
- **Very Large Files**: May require significant time/memory

### Platform
- **iOS 15 minimum**: Older devices not supported
- **Xcode required**: Cannot build without Xcode
- **PhotoKit limitations**: Restricted by iOS permissions
